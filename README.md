# Описание проекта и выбранных для тестирования взаимодействий

Проект, выбранный для лабораторной работы, представляет собой небольшое веб-приложение на Python с использованием FastAPI и SQLite, позволяющее создавать, просматривать, завершать и удалять задачи. Структура проекта включает три слоя: API, слой бизнес-логики и слой базы данных. Это позволяет изучать интеграции между различными компонентами системы: от REST-запросов до операций с базой данных, что идеально подходит для закрепления навыков интеграционного тестирования.

# Обоснование выбора этих интеграций 

Для выполнения работы были выбраны ключевые точки взаимодействия, которые покрывают реальные сценарии использования приложения. 
Среди них — создание задачи через API и запись её в базу, получение всех задач и отдельных задач по идентификатору, завершение задачи с обновлением состояния в базе и удаление задачи с последующей проверкой удаления. 
Эти интеграции важны, потому что обеспечивают правильную работу всей системы: API без корректной работы с базой данных теряет смысл, а бизнес-логика без API не имеет интерфейса для пользователя.
Выбор этих сценариев позволяет проверить именно взаимодействие компонентов, а не отдельные функции, что является основной целью интеграционного тестирования.

# Примеры кода тестов

Файл tests/test_tasks.py содержит следующие тесты:
```
def test_create_task():
    response = client.post("/tasks/", json={"title": "Test Task", "description": "Test Description"})
    assert response.status_code == 200
    data = response.json()
    assert data["title"] == "Test Task"
    assert data["completed"] is False

def test_get_tasks():
    response = client.get("/tasks/")
    assert response.status_code == 200
    assert isinstance(response.json(), list)

def test_get_single_task():
    response = client.get("/tasks/1")
    assert response.status_code == 200
    assert response.json()["id"] == 1

def test_complete_task():
    response = client.put("/tasks/1")
    assert response.status_code == 200
    assert response.json()["completed"] is True

def test_delete_task():
    response = client.delete("/tasks/1")
    assert response.status_code == 200
    assert response.json()["message"] == "Task deleted"
```

Результаты тестирования показали, что все пять интеграционных тестов успешно выполнены, что подтверждается отчётом Pytest без ошибок и с полным прохождением всех проверок. 
Это означает, что основные сценарии взаимодействия между API, бизнес-логикой и базой данных работают корректно: задачи создаются, читаются, обновляются и удаляются без сбоев.
